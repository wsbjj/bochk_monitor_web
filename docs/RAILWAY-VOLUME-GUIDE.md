# Railway 持久化存储配置指南

在 Railway 上，每次重新部署都会重置文件系统，导致日志文件丢失。为了保留日志文件，你需要配置一个 **Volume (持久化卷)**。

## 步骤 1：添加 Volume

1. 进入 Railway 项目面板，点击你的服务卡片。
2. 点击 **Volumes** 选项卡。
3. 点击 **Add Volume**（或 New Volume）。
4. 在挂载路径 (Mount Path) 输入框中填入：
   ```
   /app/data
   ```
   *注意：代码已更新，所有需要持久化的数据（日志 `data/logs` 和配置 `data/config.json`）都会统一保存在 `/app/data` 目录下。*

## 步骤 2：重新部署

添加 Volume 后，Railway 会自动触发一次重新部署。

部署完成后，`/app/logs` 目录下的所有文件都将保存在独立的持久化存储中。无论你怎么重新部署、重启服务，之前的日志文件都会一直保留。

## 验证

1. 部署完成后，运行一段时间产生一些日志。
2. 尝试重新部署一次 (Redeploy)。
3. 部署完成后，打开网页端的 "查看历史" 页面，或者使用 Railway 的 "Reference" -> "Volume" 查看文件，你应该能看到旧的日志依然存在。
